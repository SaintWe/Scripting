# 脚本配置注意事项

## 文件夹与 name 字段

**强制要求：** 脚本文件夹名称必须与 `script.json` 中的 `name` 字段完全一致。

```
scripts/
├── WeBookmarks/           ← 文件夹名
│   └── script.json        ← "name": "WeBookmarks"
├── RimeWordSync/
│   └── script.json        ← "name": "RimeWordSync"
```

## 命名规范

| 规则 | 说明 |
|------|------|
| 使用英文名称 | 避免中文或特殊字符 |
| 不含空格 | 使用驼峰命名 (CamelCase) |
| 简洁明了 | 如 `WeBookmarks`, `RimeWordSync` |

## 版本号规范

**用于自动发布流程：**

| 字段 | 说明 |
|------|------|
| `version` | 格式 `major.minor.0`，patch 必须为 0 |
| 示例 | `1.0.0`, `1.1.0`, `2.0.0` |

- 用户只修改 major 和 minor
- patch 由 GitHub Actions 自动递增
- 如果 patch ≠ 0，该脚本会被跳过

## remoteResource 配置

```json
{
  "remoteResource": {
    "url": "https://github.com/SaintWe/Scripting/releases/latest/download/脚本名.scripting",
    "hash": "",
    "autoUpdateInterval": 300
  }
}
```

| 字段 | 说明 |
|------|------|
| `url` | 指向 GitHub Releases 的下载链接 |
| `hash` | 留空，由打包流程自动填充 UUID |
| `autoUpdateInterval` | 自动更新检查间隔（秒） |
